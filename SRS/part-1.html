1. Введение

  • 1.1. Назначение

    Данный документ описывает спецификацию веб-приложения DijkstraProfi, которое визуализирует работу алгоритма Дейкстры.

  • 1.2. Область применения

    Данное веб-приложение показывает работу алгоритма Дейкстры на графе, который пользователь выбирает из предложенных. По мере работы алгоритма, состояние графа меняется, и на экран выводятся комментарии, объясняющие, что происходит на текущем шаге.

    Приложение открывается через браузер.

  • 1.3. Определения, акронимы и сокращения

    • 1.3.1. Веб-приложение - клиент-серверное приложение, в котором клиент взаимодействует с веб-сервером при помощи браузера. Логика веб-приложения распределена между сервером и клиентом, хранение данных осуществляется, преимущественно, на сервере, обмен информацией происходит по сети.

    • 1.3.2. Граф - математическая абстракция реальной системы любой природы, объекты которой обладают парными связями. Граф как математический объект есть совокупность двух множеств — множества самих объектов, называемого множеством вершин, и множества их парных связей, называемого множеством рёбер. Элемент множества рёбер есть пара элементов множества вершин.

    • 1.3.3. Алгоритм Дейкстры, далее просто алгоритм – алгоритм на графах, изобретённый нидерландским учёным Эдсгером Дейкстрой в 1959 году. Находит кратчайшие пути от одной из вершин графа до всех остальных. Алгоритм работает только для графов без рёбер отрицательного веса.

    • 1.3.4. DijkstraProfi – веб-приложение для визуализации алгоритма Дейкстры.

    • 1.3.5. start – начальная вершина в графе. Алгоритм ищет кратчайшие пути от неё до всех остальных.

    • 1.3.6. distance(start, v) – длинна кратчайшего из (известных нам на данный момент) путей от начальной вершины start до вершины v, где v – произвольная вершина графа. По ходу работы алгоритма значения в distance будут обновляться. Изначально distance(start, v) = 0, а для всех остальных вершин длинна пути равна бесконечности.

    • 1.3.7. Обработанными будем называть вершины, для которых distance больше не будет обновляться. То есть те, для которых мы уже нашли кратчайший возможный путь. В начале, конечно, все вершины необработанные.
